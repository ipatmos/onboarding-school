# 신입 간사 온보딩 스쿨

팻머스 신입 간사의 온보딩 훈련 과정을 관리하는 정적 웹앱입니다. 브라우저에서 바로 실행할 수 있고, GitHub Pages에 올려서 공유할 수 있습니다.

## 주요 기능

- 계정별 로그인: 신입 간사와 관리자 화면 분리
- 신입 간사 관리: 등록, 수정, 삭제
- 훈련 과정 관리: 추가, 복사, 수정, 삭제, 드래그 순서 변경
- 과정별 제출: 요약, 질문, 체크 확인, 드라이브 업로드 안내
- 제출물 검토: 통과, 보완 요청, 반려, 삭제
- 영상 과정: 유튜브/비메오 링크 첫 화면 표시, 여러 영상 링크 지원
- 진행률: 착륙 장면 형태의 진행률 시각화
- 팻머스 로고 적용

## 실행 방법

1. 이 폴더의 `index.html`을 브라우저로 엽니다.
2. 로그인 화면에서 계정을 선택합니다.
3. 신입 간사는 `나의 온보딩`, `훈련 과정`, `제출함`을 사용합니다.
4. 관리자는 `관리자` 화면에서 간사, 훈련 과정, 제출물을 관리합니다.

## 기본 로그인

- 관리자 비밀번호: `admin123`
- 기본 신입 간사: `나현영` / 비밀번호: `0000`

공개 저장소에 올리는 프로토타입이므로 실제 운영 전에는 관리자 비밀번호와 데이터 저장 방식을 서버 기반으로 바꾸는 것이 좋습니다.

## GitHub에 올릴 파일

다음 파일과 폴더를 올리면 됩니다.

- `index.html`
- `styles.css`
- `app.js`
- `assets/`
- `README.md`
- `.gitignore`

`.codex-*.ps1` 파일은 로컬 미리보기용 임시 파일이라 `.gitignore`에 제외했습니다.

## GitHub Pages 배포

1. GitHub에 새 저장소를 만듭니다.
2. 이 폴더의 파일을 저장소에 업로드합니다.
3. 저장소의 `Settings` → `Pages`로 이동합니다.
4. `Deploy from a branch`를 선택합니다.
5. Branch를 `main`, Folder를 `/root`로 선택하고 저장합니다.
6. 잠시 후 제공되는 Pages 주소로 접속합니다.

## 데이터 저장 방식

현재 버전은 서버가 없는 정적 프로토타입입니다. 간사 정보, 훈련 과정, 제출물, 진행률은 접속한 브라우저의 `localStorage`에 저장됩니다. 따라서 다른 컴퓨터나 다른 브라우저와 데이터가 자동 공유되지는 않습니다.

실제 여러 사람이 함께 사용하는 운영 버전으로 확장하려면 서버와 데이터베이스, 인증 시스템이 필요합니다.
